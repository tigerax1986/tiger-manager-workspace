## Stadium Maintenance – Seat Grid UI Boost (April 23, 2026 — continued)

### User Request #1 — Row spacing & canvas improvements
- Row definitions editor: move buttons next to "To Seat" box, swap cascade/bin order, align bin icons vertically
- Compact page headers across all 6 admin pages
- Visual gaps in row grid when sequences are non-consecutive (e.g. 1, 2, 5 → spacer rows for 3, 4)
- Horizontal scrolling for seat rows (no wrapping), grab-to-pan canvas
- Edit Canvas toggle (mode-gated: seat offset or canvas panning)
- Row reorder via drag handles left of sequence numbers

### Implementation (Batch 1)
- **Row definitions button layout**: Removed `margin-left: auto` from `.row-actions-col`, added `min-width: 56px` for vertical bin alignment. Cascade (📋) initially left of bin (🗑️), swapped back on user request.
- **Spacer rows** (`gridRows` computed): Fills all integers between min and max sequence. Spacer rows get `isEmpty: true`, semi-transparent with dashed pattern, no interactive elements.
- **Horizontal scroll**: `.row-seats` changed from `flex-wrap: wrap` to `nowrap`. Seats use `flex-shrink: 0`.
- **Grab-to-pan**: `onCanvasMouseDown/Move/Up` functions. Cursor switches `grab`/`grabbing`. Only when Edit mode OFF.
- **Edit Canvas toggle**: `🖐️ Edit Canvas` button toggles `seatDragMode`. When ON: seat cursor = `move`, seat drag fires, row drag handles draggable. When OFF: canvas panning.
- **Row reorder**: `⠿` drag handle left of sequence number. HTML5 drag-and-drop (`onRowDragStart`, `onDragOver`, `onDrop`, `onRowDragEnd`). Dropped rows recalculate sequences (top = highest), calls `seatService.update()` per row. Drop target shows blue dashed outline.
- **CSS compilation fixes**: Stray fragments in EmailDesigner, CustomerRegistration, CustomerMaintenance resolved.

### User Request #2 — Panel cleanup, sequence sizing, spacer drag fix
- Remove "Seat Grid Editor" panel header + merge its buttons into canvas-header toolbar
- Make sequence edit box smaller (was overlapping row label)
- Fix drag into spacer rows not working

### Implementation (Batch 2)
- **Panel removal**: Deleted entire `.panel-header` div with h3 + buttons. Merged Save Changes, Refresh, Add Rows, 🖐️ Edit, Edit Seats, Edit Seq/Update/Cancel into canvas-header.
- **Sequence sizing**: `.row-sequence` width from 60px → 44px, input width to 40px, compact padding + smaller font.
- **Spacer drop fix**: Removed `pointer-events: none` from `.row-spacer`. When dropped on a spacer, the dragged row takes that sequence number; old sequence becomes the new gap. `onDrop` handles both non-empty and empty target paths.

### User Request #3 — Grid header alignment & canvas zoom
- "Seats" text not in line with starting seat; row label too close to sequence
- Canvas grab-to-pan and zoom out to show empty space for future draw-seats function

### Implementation (Batch 3)
- **Grid header alignment**: Added hidden `.row-drag-handle` in grid-header to match row structure. Removed `padding-left` from Seats header label. Row label column widened to 90px, right-aligned, pushed toward seats area.
- **Canvas zoom**: Ctrl+wheel zoom initially, then changed to wheel-only zoom (no modifier). Data-driven auto-fit on area load: calculates total grid dimensions from `maxCols × seatWidth + gaps + fixed columns`, sets `canvasZoom` to fit canvas viewport with 5% margin. Range 0.3–2x. Grid padding reduced from `100px 50px` to `10px 4px`.

### Key Implementation Details
- **Canvas overflow hidden when zooming**: `.canvas` uses `overflow: hidden` + transform on `.grid-render` with `transformOrigin: top left`
- **Zoom cursor tracking**: Scroll adjusts so the point under cursor stays centered
- **Panning**: Global mousemove/mouseup listeners; cursor styles switch on drag
- **Drag-to-offset vs panning conflict resolution**: `seatDragMode` boolean gates which behavior triggers
- **Spacer row interaction**: No `pointer-events: none` (blocks HTML5 drag target), reduced opacity (0.4), no hover effects
- **Row reorder with HTML5 DnD**: `draggable="true"` on handle, filters non-empty rows, recalculates sequences descending
- **Auto-zoom data-driven**: Dimensions derived from `seats` array, not DOM measurement — avoids layout timing issues with transforms
