### UX Fixes (iteration 2 - this session)
- **Cascade logic**: `*ALL` at any level hides child fields
- **Row multi-select**: `<select multiple>` populated from actual DB via `/api/rows`
- **Area → auto-select `*ALL` rows** by default
- **Seat multi-select**: `<select multiple>` populated from DB via `/api/seat-numbers`, shown only when exactly one row selected (not `*ALL`)
- **`confirm()` → proper delete modals** for price code, tier, and mapping row deletion
- **Mapping rendering bug**: `price_code` was being caught by normalization (starts with `price_`), uppercased nonexistent `price_CODE` → all `m.price_code` returned undefined → no mappings showed. Fixed: only normalize suffix if it's a single alphanumeric char (A-Z or 0-9)
- **API endpoint**: Added `/api/rows` (distinct row codes per stadium/stand/area) and `/api/seat-numbers` (distinct seat numbers per stadium/stand/area/row)

## Decisions
- Row/seat selectors must be DB-driven (validation + consistency), never free-text
- Multi-select for rows: Ctrl+click for multiple, `*ALL` top option
- Multi-select for seats: same pattern, shown only for single-row selection
- Delete confirmations are now proper modals, not browser dialogs
- When area is selected → auto-default rows to `*ALL`
- When multiple rows selected → no seat section shown (all `*ALL` seats)
- When single row selected → show seat multi-select for further granularity

## Open Issues
- Multi-seat creation saves one mapping per seat — but if a seat mapping already exists (conflict), it just updates tier_description. Need to verify this doesn't cause unintended overwrites.
- Need to test the full workflow in browser end-to-end.

## Price Tiers Refactoring

**Decision**: Move pricing from per-mapping-row to tier-level via a new `price_tiers` table.

### Migration: `price_mappings_link_tier.001.sql`
- Added `tier_id UUID REFERENCES price_tiers(id)`
- Backfilled tier_id by matching `(stadium_code, product_code, price_code, tier_description)` against `price_tiers`
- Dropped: `stadium_code`, `product_code`, `price_code`, `tier_description` columns from `price_mappings`
- Set `tier_id NOT NULL` after backfill
- New UNIQUE: `(tier_id, stand_code, area_code, row_code, seat_number)`

### API Updates
- `GET /api/price-mappings` — JOINs with `price_tiers` to include all 36 price band columns AND `tier_id`
- `POST /api/price-mappings` — accepts `tier_id` instead of `(stadium_code, product_code, price_code, tier_description)`
- `POST /api/price-mappings/batch-save-tier` — accepts `tier_id` instead of composite key
- `PUT /api/price-mappings/:id` — only accepts `(stand_code, area_code, row_code, seat_number)` — tier changes require DELETE + re-add
- `GET /api/price-tiers` — fetches tier-level pricing
- `PUT /api/price-tiers` — upsert by `(stadium_code, product_code, price_code, tier_description)`
- `PUT /api/price-tiers /500 fix`: `updated_at = NOW()` was being appended inside the `.map()` for every price column, causing "multiple assignments to same column" error. Fixed by adding `updated_at = NOW()` once after the `UPDATE SET` clauses join.

### Frontend Updates
- **Price display**: Prices shown on tier header (click the tier name/description area) — not on individual mapping rows
- **No "Set Prices" button**: Clicking the tier header opens the band editor modal
- **`openBandEditor`**: Now takes `(tier, pc)` instead of `(mapping, pc)`, reads prices from `mapping[0]` (which includes JOINed price columns)
- **`saveBandPrices`**: PUTs to `/api/price-tiers`, saving at tier level
- **Normalisation fix**: `fetchAllMappings` and `fetchTierPrices` normalise Postgres lowercase price column keys (`price_a` → `price_A`) to match band codes
- **State managed**: `shouldNormalize` flag in `fetchAllMappings` controls case conversion
- **Backfill script**: `backfill_tiers.001.sql` created missing tier records for existing mappings that had no matching `price_tiers` entry

### Backfill Issue
- Initial migration failed because existing mappings referenced tier descriptions (Silver, Bronze) that didn't have `price_tiers` records yet
- Created `backfill_tiers.001.sql` to INSERT missing tier records based on DISTINCT mapping data
- Ran with `run_tier_migration.mjs` script (node-pg exec)
- Fixed obsolete FK column name (`price_tiers.stadium_code` → `price_tiers(stadium_code)` since `stadiums(stadium_code)` is the PK)
