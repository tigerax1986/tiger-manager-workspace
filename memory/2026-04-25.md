### Seat Deletion Offset Fix (In Progress)
- **Bug**: When deleting the first seat(s) in a row, remaining seats' offsets weren't preserved — seat 4 would visually move to position 1 instead of staying at position 3
- **Root cause**: The offset recalculation loop started at `i = 1`, skipping the first remaining seat entirely
- **Fix**: Added handling for `i = 0` — for the first remaining seat in each affected row, calculate its new offset as:
  - `sum(1 + deleted_seat.offset_x) for all deleted seats before this seat + existing offset_of_this_seat`
  - This accounts for the visual elements (seat + its trailing offset dummies) that were removed to the seat's left
