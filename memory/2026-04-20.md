## Multi‑Select Shapes Feature (01:02 AM)

### User Request:
- **Multi‑select shapes** with Ctrl/Cmd+click
- **Move multiple shapes** together (mouse drag or arrow keys)
- **Delete multiple shapes** at once
- **Property editing** disabled for multi‑selection (only moving allowed)

### Implementation:

#### Data Model:
- Added `selectedShapes` ref (Shape[]) to track multiple selections
- Maintained `selectedShape` as primary selection (last clicked)
- Updated all selection‑related logic to handle arrays

#### Selection Behavior:
- **Ctrl/Cmd+click**: Toggles shape in/out of selection
- **Click without Ctrl**: Replaces selection with single shape
- **Canvas click (no shape)**: Clears selection unless Ctrl held

#### Drag & Move:
- **Mouse drag**: Moves all selected shapes if clicked shape is in selection, otherwise just clicked shape
- **Arrow keys**: Nudges all selected shapes by 1 pixel (or primary shape if none selected)
- **Position tracking**: Uses `dragOriginalPositions` Map to store initial positions for each shape

#### Property Panel:
- **Multiple selection**: Shows warning message “Multiple shapes selected (N)”
- **Single selection**: Shows full property editor as before
- **Property‑actions**: Duplicate/Delete buttons remain visible (work on selection)

#### Visual Feedback:
- All selected shapes highlight with `.selected` class (blue stroke)
- Selection count displayed in canvas header

#### Deletion:
- `deleteShape()` now deletes all selected shapes
- Works with both single and multi‑selection

#### Key Changes:
1. `selectShape()` – Ctrl‑aware toggle logic
2. `startDrag()` / `handleDrag()` – multi‑shape movement
3. `handleKeydown()` – arrow keys move all selected shapes
4. `deleteShape()` – delete entire selection
5. Template – updated `:class` bindings and selection display

## Multi‑Select Bug Fixes & SVG Load Tool Default (03:15 AM)

### Issues Reported:
1. **Multi‑selection stops working after moving any shape once**
2. **Default tool on load is rectangle; should switch to select tool when SVG loads**

### Root Causes Identified:
1. **Stale drag references**: `dragSelectedShapes` and `dragOriginalPositions` not cleared after drag, causing interference with subsequent selection logic
2. **Ctrl‑key ignored in drag selection**: `startDrag` was updating selection even when Ctrl/Cmd pressed, conflicting with `selectShape` toggle logic
3. **Zombie shape references**: Loading new SVG layout didn't clear `selectedShapes`, leaving references to old shape objects no longer in `shapes` array

### Fixes Applied:

#### 1. Clear Drag State in `stopDrag()`:
```javascript
dragSelectedShapes.value = []
dragOriginalPositions.value.clear()
dragOriginalX.value = 0
dragOriginalY.value = 0
dragOriginalPoints.value = []
```

#### 2. Respect Ctrl Key in `startDrag()`:
- Added `&& !event.ctrlKey && !event.metaKey` condition to selection update
- Prevents `startDrag` from replacing selection when Ctrl/Cmd is held
- Lets `selectShape` handle the toggle logic exclusively

#### 3. Clear Selection on SVG Load:
- Added `selectedShapes.value = []`, `selectedShape.value = null`, `shapeImageUrl.value = ''` in `parseSvgData()`
- Prevents zombie references to old shape objects

#### 4. Default Tool Switch:
- Added `activeTool.value = 'select'` after `parseSvgData()` in `loadSvgLayout()`
- Ensures editing mode (select tool) is active when viewing loaded SVG layouts

### Commit Reference:
- `ticketing‑system`: commit `90b79ea` – Multi‑select bug fixes and SVG load improvements

### Expected Behavior After Fixes:
- **Multi‑selection continues to work after moving shapes**
- **Ctrl+click toggles selection consistently**
- **Loading SVG layout automatically switches to select tool**
- **No stale references interfering with selection/dragging**
